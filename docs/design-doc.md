# 街机乒乓球游戏 - 设计文档

## 一、场景与节点树方案

### 1.1 整体架构

```
Game (游戏主控制器)
├── Ball (球)
│   ├── trail[] (拖尾轨迹点数组)
│   └── onFire (火焰状态)
├── Paddle[2] (球拍 x2)
│   ├── isSmashing (扣杀动画状态)
│   └── smashCooldown (扣杀冷却)
├── AIOpponent (AI对手)
│   ├── difficulty (难度系数)
│   └── predictedY (预测落点)
├── EffectsManager (特效管理器)
│   ├── particles[] (粒子系统)
│   ├── screenShake (屏幕震动)
│   └── flashColor (屏幕闪光)
└── UIManager (UI管理器)
    ├── startMenu (开始菜单)
    ├── pauseMenu (暂停菜单)
    ├── gameOverMenu (结束菜单)
    ├── restCountdown (局间休息)
    └── gameHUD (游戏HUD)
```

### 1.2 Canvas 分层渲染

游戏使用单 Canvas 2D 渲染，按以下顺序绘制：

1. **背景层**: 球台、球网、边框
2. **游戏对象层**: 球拍、球、拖尾
3. **特效层**: 粒子效果、屏幕震动、闪光
4. **UI层**: HTML DOM 元素（菜单、计分板）

### 1.3 游戏状态机

```
menu (菜单)
  ↓ startGame
playing (游戏中) ←→ scored (得分动画)
  ↓ ESC / pauseBtn
paused (暂停)
  ↓ resume
playing
  ↓ 达到11分且领先2分
rest (局间休息，5秒倒计时)
  ↓ 倒计时结束
playing (新一局)
  ↓ 先赢2局
gameOver (游戏结束)
```

## 二、物理与操控系统说明

### 2.1 球的物理模型

#### 运动公式
```
vy += gravity        // 重力加速度
x += vx * speedMult  // x方向位移
y += vy * speedMult  // y方向位移
```

#### 参数配置
| 参数 | 值 | 说明 |
|------|-----|------|
| `GRAVITY` | 0.15 | 每帧重力加速度 |
| `BOUNCE_DAMPING` | 0.85 | 上下边界反弹阻尼系数 |
| `INITIAL_SPEED_X` | 5 | 初始x方向速度 |
| `SPEED_INCREMENT` | 0.03 (3%) | 每板球速增幅 |
| `MAX_SPEED` | 15 | 最大球速限制 |

#### 球速倍率机制
- 每次击球后 `speedMultiplier *= 1.03`
- 扣杀时额外乘以 `1.4`
- 封顶速度由 `MAX_SPEED` 限制
- 得分后重置为 1.0

### 2.2 球拍操控系统

#### 加速度模型
```
按下按键:   vy += ACCELERATION
松手后:     vy *= FRICTION (0.9)
最大速度:   |vy| <= MAX_SPEED (10)
```

#### 参数配置
| 参数 | 值 | 说明 |
|------|-----|------|
| `ACCELERATION` | 0.8 | 每帧加速度 |
| `MAX_SPEED` | 10 | 最大移动速度 |
| `FRICTION` | 0.9 | 惯性摩擦系数 |

#### 按键映射
| 按键 | 玩家1 | 玩家2 |
|------|-------|-------|
| 上移 | W | ↑ |
| 下移 | S | ↓ |
| 扣杀 | D | L |

### 2.3 碰撞检测

采用 **圆形-矩形最近点** 碰撞算法：

```
1. 找到矩形上距离球心最近的点 (closestX, closestY)
2. 计算球心到该点的距离 distance
3. 若 distance < radius 则发生碰撞
4. 返回碰撞点相对球拍中心的 y 比例 (-1 ~ 1)
```

击球角度由碰撞点决定：
- 击中球拍上部 → 球向上飞
- 击中球拍中部 → 球水平飞  
- 击中球拍下部 → 球向下飞

### 2.4 扣杀机制

#### 触发条件
1. 球处于**高点区域**（球台中部 ±75px 范围内）
2. 球与球拍距离 < 80px
3. 扣杀冷却时间已过（500ms）

#### 扣杀效果
- ✅ 球速提升 **40%**
- ✅ 球方向变窄（垂直速度减半）
- ✅ 触发屏幕震动 + 闪光特效
- ❌ 失败则挥空（无惩罚，正常漏球）

## 三、AI难度调节算法

### 3.1 核心机制

AI 难度通过三个维度动态调节：

| 维度 | 公式 | 范围 |
|------|------|------|
| 反应时间 | `200 - 150 * difficulty` | 50ms ~ 200ms |
| 击球精度 | `0.7 + 0.25 * difficulty` | 70% ~ 95% |
| 扣杀概率 | `0.2 + 0.5 * difficulty` | 20% ~ 70% |

### 3.2 落点预测算法

AI 使用**模拟推演**预测球的落点：

```javascript
function predictBallY(ball, targetX) {
    let simX = ball.x, simY = ball.y;
    let simVx = ball.vx, simVy = ball.vy;
    
    while (simX 未到达 targetX) {
        simVy += gravity;
        simX += simVx;
        simY += simVy;
        
        // 处理上下边界反弹
        if (碰到边界) simVy = -simVy * damping;
    }
    
    return simY;
}
```

### 3.3 动态难度调整

#### 调整规则
- 玩家领先 → AI 变强（难度 +0.05 × 分差）
- 玩家落后 → AI 变弱（难度 -0.05 × 分差）
- 每局开始时根据局分差调整基础难度

#### 难度边界
- 最低难度: 0.1（非常简单，反应慢、经常打偏）
- 最高难度: 1.0（专业级，几乎不失误）

## 四、特效与动画配置

### 4.1 对拉火焰特效

#### 触发条件
连续对拉 **8板** 及以上

#### 视觉效果
- 球拖尾粒子（15个轨迹点）
- 火焰渐变色: 红 → 橙 → 黄
- 粒子大小随距离衰减
- 透明度渐变

#### 颜色配置
```javascript
FLAME_COLORS = [
    '#ff6b00',  // 深橙红
    '#ffaa00',  // 橙色
    '#ffff00',  // 亮黄
    '#ff4400'   // 红色
]
```

### 4.2 粒子系统

#### 击球粒子
- 数量: 8个（扣杀时16个）
- 寿命: 30~50帧
- 速度: 2~6
- 颜色: 白、黄、橙（扣杀时火焰色）

#### 得分粒子
- 数量: 20个
- 向四周爆炸扩散
- 颜色: 对应玩家颜色

### 4.3 屏幕反馈

| 事件 | 震动强度 | 闪光颜色 |
|------|----------|----------|
| 普通击球 | 0 | 无 |
| 扣杀 | 8px | #ff6b00 (30%) |
| 得分 | 5px | 无 |
| 游戏结束 | 10px | 胜者颜色 (40%) |

### 4.4 动画配置

#### 扣杀动画
- 持续时间: 200ms
- 球拍放大: 宽度 ×1.67, 高度 ×1.25
- 发光效果: 橙色光晕

#### 分数动画
- 对拉计数 ≥3 时显示
- 每次增加触发脉冲动画
- 渐变显示/隐藏

## 五、游戏规则

### 5.1 计分规则
- 先到 **11分** 赢一局
- 必须领先 **2分** 以上
- **三局两胜** 制

### 5.2 局间休息
- 每局结束后休息 **5秒**
- 显示倒计时和当前局分
- 休息结束自动开始下一局

### 5.3 发球规则
- 每分由得分方发球
- 球从发球方一侧发出
- 初始方向朝向对手
